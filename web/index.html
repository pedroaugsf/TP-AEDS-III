NutriTrack

Site mínimo para gerenciar valor nutricional consumido (CRUD em arquivos binários).

Usuários

CRUD de usuários (nome, email, data de nascimento e telefones).

  • Campos: string, data, string multivalorada (telefones)
  • Arquivo: ./dados/usuario/usuario.db

Alimentos

CRUD de alimentos (nome, marca, macros e tags).

  • Campos: string, float, string multivalorada (tags)
  • Arquivo: ./dados/alimento/alimento.db

Refeições

CRUD de refeições (data, tipo, observação).

  • Campos: data, string
  • Arquivo: ./dados/refeicao/refeicao.db

Consumos

Tabela intermediária (N:N) entre Refeição e Alimento.

  • Campos: FK refeicaoId, FK alimentoId, float (gramas)
  • Arquivo: ./dados/consumo/consumo.db

Como testar

  1. Compile: javac -d out src/app/**/*.java
  2. Execute: java -cp out app.Main
  3. Veja os arquivos binários gerados em ./dados

Esta interface é propositalmente mínima (HTML/CSS). A lógica está em Controller/DAO.

Projeto exemplo — MVC + DAO (arquivo binário com cabeçalho e lápide).
